# PRD：网盘同步观影房间 v1

## Problem Statement

已经通过 AList 或 OpenList 挂载网盘资源的用户，希望和少量朋友一起同步观看同一个网盘视频，并在观看时实时语音交流。用户不需要媒体库、转码、账号体系或复杂权限，只需要快速选片、建房、加入、同步播放和语音。

## Solution

提供一个轻量 Web 应用：用户创建或加入最多 3 人的临时房间，并在创建时选择观影模式。直链同步模式下，后端安全连接 AList/OpenList，前端浏览白名单目录并选择视频，成员各自播放同一视频直链并同步状态。房主推流模式下，房主选择本地视频文件，不上传后端，只在浏览器本地播放并通过 WebRTC 推流给观众。房间状态保存在内存中，适合单机部署。

## User Stories

1. 作为观影用户，我想输入或随机生成昵称，以便用临时身份加入房间。
2. 作为观影用户，我想浏览已授权的网盘目录，以便选择要看的视频。
3. 作为观影用户，我想创建空房间，以便先邀请朋友再决定看什么。
4. 作为观影用户，我想通过房间码或邀请链接加入房间，以便快速进入观影会话。
5. 作为房间成员，我想在需要时输入房间密码，以便加入受保护的房间。
6. 作为房间成员，我想选择或更换当前视频，以便房间一起观看新的资源。
7. 作为房间成员，我想播放、暂停、拖动进度和修改倍速，以便所有人保持同步。
8. 作为新加入成员，我想自动同步当前视频、字幕和播放位置，以便不用手动追进度。
9. 作为房间成员，我想选择同目录字幕，以便大家看到同一份字幕。
10. 作为房间成员，我想加入语音、静音、开麦和调节语音总音量，以便观看时沟通。
11. 作为房间成员，我想刷新页面后恢复身份和房间状态，以便短暂断线不中断观影。
12. 作为部署者，我想通过环境变量配置 AList/OpenList 和 STUN/TURN，以便安全部署。
13. 作为房主，我想创建房主推流房间，以便像私人直播间一样把本地视频播放给朋友看。
14. 作为房主，我想选择本机视频文件并直接推流，以便不用上传视频或配置网盘。
15. 作为观众，我想进入房主推流房间后观看房主推出来的画面和声音，以便不需要访问原始视频文件。
16. 作为观众，我想请求播放、暂停、跳转或调整倍速，以便让房主端自动执行观看控制。
17. 作为房主，我想在离开时直接结束推流房间，以便不会留下无法继续播放的空直播间。

## Implementation Decisions

- 使用单仓双应用：Vue 3 + Vite 前端、NestJS + TypeScript 后端、共享类型包。
- AList/OpenList 固定 Token 只保存在后端环境变量中，前端不接触 Token。
- 启用 `ALLOWED_ROOT_PATHS` 白名单，后端拒绝越界路径。
- 房间状态使用内存 Map，房间人数上限 3 人。
- 创建房间可选加入密码，后端只保存哈希。
- 所有成员都可播放控制和换片，服务端按接收顺序写入递增版本号。
- 房主离线 60 秒后自动按加入顺序转让；空房间 60 秒后释放。
- 字幕从视频同目录手动选择，支持 VTT 直出和 SRT 转 VTT。
- 播放同步和 WebRTC 信令复用同一个 WebSocket。
- WebRTC 使用 P2P mesh，默认 STUN，TURN 可选配置。
- 新增 `direct` 和 `host-stream` 双模式，创建房间时确定模式，不支持房间内切换。
- 房主推流模式中，本地视频文件不上传后端，后端只维护推流状态和转发 WebRTC 信令。
- 房主推流模式不做独立字幕传输；如果需要字幕，由房主本地播放器画面承载。
- 房主推流模式中观众控制会转发为房主端命令，默认自动执行。

## Testing Decisions

- 测试优先覆盖后端核心行为，测试公开服务接口和可观察行为，不测试私有实现细节。
- AList 模块测试白名单、视频过滤、错误转换。
- 房间模块测试创建、加入、密码、人数上限、重连、房主转让、空房释放。
- 播放同步测试版本号后到为准和完整状态同步。
- 字幕模块测试同目录字幕、VTT 直出和 SRT 转 VTT。
- 语音测试信令转发和状态广播。
- 房主推流模式测试创建状态、开播/停播、控制请求转发、房主离开关闭房间、推流信令目标转发。

## Out of Scope

v1 不做账号系统、聊天、播放列表、历史记录、收藏、视频代理、转码、媒体刮削、多实例部署、SFU 语音、服务端音频录制和混流。房主推流模式不做本地文件上传、服务端录制、服务端转码、媒体服务器中转和独立字幕传输。

## Further Notes

公网部署时应启用 HTTPS，因为浏览器麦克风权限通常要求安全上下文。视频直链播放依赖 AList/OpenList 返回的 URL 可被浏览器直接访问；如果后续遇到跨域或鉴权问题，再新增视频代理模式。
